MGAP Search API
POST /api/search

Поиск персон через обращение к API Pipl.com
Поддерживает 2 варианта поиска: по набору поисковых параметров или по searchPointer. Переход по searchPointer должен использоватеься при переходе из списка возможных совпадений. Чтобы не снималась оплата в запрос всегда добавляется: clark.kent@example.com и возвращается одна и тажа запись.

Запрос: https://docs.pipl.com/reference#person
Ответ: https://docs.pipl.com/reference#response

Пример:
curl --location --request POST 'http://search.mgap.ru:8081/api/search' \ --header 'Content-Type: application/json' \ --data-raw '{"names":[{"first":"Kirill","last":"Lazarenko"}],"addresses":[{"raw":"Moscow, Russia"}],"phones":[],"emails":[{"address":"clark.kent@example.com"}]}'

GET /api/getPerson

Возвращает персону из локальной базы данных по идентификатору personId

Запрос: personId - идентификатор персоны
Ответ: SearchResponse + personId

GET /api/refreshPerson

Обновляет сохраненную персону через вызов к API Pipl.com

Запрос: personId - идентификатор персоны
Ответ: SearchResponse + personId
